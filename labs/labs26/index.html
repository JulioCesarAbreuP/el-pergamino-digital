Lab 26 – Evaluación de Hardenización de Sistemas Linux y Buenas Prácticas

Aplicación de técnicas de hardening para reforzar seguridad en entornos Linux

Objetivo

Este laboratorio se centra en aplicar y verificar mejores prácticas de hardening en sistemas Linux, reduciendo la superficie de ataque y reforzando configuraciones de seguridad.

Alcance y limitaciones

Solo en entornos controlados. No realizar pruebas de hardening en sistemas productivos sin respaldo y autorización.

Requisitos previos

  • Conocimientos de administración de Linux
  • Acceso a consola de sistema
  • Herramientas como Lynis, auditd, ufw

Entorno de laboratorio

Sistema Linux base (Ubuntu/Debian/CentOS) con usuario root o sudo para aplicar hardening.

Desarrollo

Aplicar recomendaciones de CIS Benchmark, activar firewall, revisión de servicios y auditoría.

Resultados esperados

  • Servicios innecesarios deshabilitados
  • Firewall activo y seguro
  • Permisos de archivos ajustados

Análisis de resultados

Comparar antes y después del hardening. Detectar brechas persistentes y riesgos mitigados.

Medidas

  • Auditoría con Lynis
  • Aplicar RBAC
  • Restricción de usuarios con sudo

Contramedidas

  • Deshabilitar servicios innecesarios
  • Configurar firewall de manera restrictiva

Riesgos y advertencias

Hardening inadecuado puede bloquear servicios legítimos. Siempre respaldar antes de aplicar cambios.

Evidencias

Espacio reservado para capturas de comandos, logs y reportes.

Validación cruzada

  • MITRE ATT&CK – Hardening Checklist
  • CIS Controls – Secure Configuration
  • NIST SP 800-123 – Guide to General Server Security

Certificaciones asociadas

Security+, GSEC, eJPTv2

Glosario asociado

Hardening, CIS Benchmark, firewall, auditoría

Preguntas de refuerzo

¿Qué pasos reduce la superficie de ataque? ¿Cómo auditar servicios activos?

Rúbrica de evaluación

  • ✔ Hardening aplicado
  • ✔ Auditorías realizadas

Recursos complementarios

  • CIS Linux Benchmark
  • Lynis – Audit tool
