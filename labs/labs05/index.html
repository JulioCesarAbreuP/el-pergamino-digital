Lab 05 – Escaneo de vulnerabilidades con OpenVAS

Detección automatizada de vulnerabilidades en sistemas mediante escaneo estructurado con Greenbone/OpenVAS

Objetivo

Aprender a utilizar OpenVAS para realizar escaneos de vulnerabilidades en sistemas objetivo, interpretar los resultados, priorizar riesgos y generar reportes técnicos como parte de una estrategia de gestión de vulnerabilidades.

Alcance y limitaciones

Alcance: Instalación y uso de OpenVAS, escaneo de objetivos vulnerables, análisis de resultados, exportación de reportes.

Limitaciones: No se realiza explotación de vulnerabilidades. El escaneo se limita a entornos de laboratorio controlado.

Máquina vulnerable sugerida: Metasploitable 2 (descarga oficial)

Requisitos previos

  • Conocimientos básicos de redes y servicios comunes
  • Familiaridad con entornos Linux y herramientas de escaneo
  • Entorno: Kali Linux o Greenbone Community Edition (GVM), Metasploitable2
  • Recursos: 4 GB RAM, 2 vCPU, conexión de red funcional

Arquitectura del entorno

[ Kali/GVM Scanner (192.168.1.10) ] ←→ [ Router ] ←→ [ Metasploitable2 (192.168.1.20) ]

Ambas máquinas deben estar en la misma red virtual o física, con visibilidad mutua para el escaneo.

Herramientas y entorno técnico

  • OpenVAS / GVM: motor de escaneo de vulnerabilidades
  • Greenbone Security Assistant (GSA): interfaz web para gestión de escaneos
  • Metasploitable2: máquina vulnerable para pruebas
  • Wireshark (opcional): validación del tráfico de escaneo

Procedimiento técnico paso a paso

  1. Actualizar repositorios y dependencias: 
    sudo apt update && sudo apt install openvas -y
  2. Inicializar OpenVAS: 
    sudo gvm-setup
  3. Verificar estado del servicio: 
    sudo gvm-check-setup
  4. Iniciar interfaz web: 
    sudo gvm-start

    Acceder vía navegador: https://127.0.0.1:9392

  5. Iniciar sesión con usuario admin y contraseña generada
  6. Crear nuevo objetivo: IP de Metasploitable2 (ej. 192.168.1.20)
  7. Crear tarea → seleccionar objetivo → iniciar escaneo
  8. Esperar finalización y revisar resultados
  9. Exportar reporte en HTML o PDF

Resultados esperados

  • Listado de vulnerabilidades detectadas en el sistema objetivo
  • Clasificación por severidad (baja, media, alta, crítica)
  • Asociación con CVEs, vectores de ataque y soluciones sugeridas

Análisis de resultados

El escaneo revela múltiples vulnerabilidades en servicios como FTP, Apache, Samba y MySQL. Se identifican CVEs con severidad crítica, incluyendo vulnerabilidades de ejecución remota de código y escalada de privilegios. El reporte permite priorizar acciones correctivas basadas en el riesgo.

Medidas

  • Documentar vulnerabilidades detectadas y sus CVEs
  • Clasificar por criticidad y facilidad de explotación
  • Planificar parches, actualizaciones o mitigaciones

Contramedidas

  • Actualizar servicios vulnerables a versiones seguras
  • Aplicar parches de seguridad recomendados
  • Deshabilitar servicios innecesarios
  • Implementar segmentación de red y control de acceso

Riesgos y advertencias

El escaneo de vulnerabilidades puede generar tráfico intenso y activar alertas en sistemas IDS/IPS. No ejecutar OpenVAS en redes de producción sin autorización. El uso indebido puede ser considerado actividad hostil.

Reflexión estratégica

El escaneo de vulnerabilidades es una piedra angular en la defensa proactiva. OpenVAS permite identificar debilidades antes de que sean explotadas, pero su verdadero valor reside en la capacidad de interpretar los hallazgos, priorizar riesgos y actuar con criterio. La seguridad no es un estado, es un proceso continuo.
